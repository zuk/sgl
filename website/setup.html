古いsgl設定方法

ここにあげているのは、以前のsglの設定方法です。 古い情報であり、現在のversionには適用できません。

●ダウンロード

●コマンドプロンプトの使い方
・マイ コンピュータをダブルクリック
・「'debian' の pub(Y:)」をダブルクリック
・「コマンドプロンプト」をデスクトップにドラッグしてコピーする。
(すでにファイルが存在する場合は上書きする。)
・デスクトップにできた「コマンドプロンプト」をダブルクリックして立ち上げ。
ウインドウタイトルのところに"Ruby Prompt"というのが出てきたらOK。

●自宅でsglを使いたい場合

以下のファイルをダウンロードし、c:\rubyとなるように解凍してください。

  • Ruby一式 まずこれを入れてください。
  • sgl単体 その上にこのsglを入れてください。
  • bass.dll サウンド用のdll
  • xyzzy Ruby対応のエディター

解凍したRuby一式の中にすでにsite_rubyというディレクトリーがありますが、 それと重なるように上書きしてください。つまり、 c:\ruby\lib\ruby\site_rubyとなるように上書きしてください。

以下のPATHの設定方法を参考にしてc:\ruby\binにPATHを通してください。

PATHを通したらコマンドプロンプトを立ち上げて、
c:\> ruby -v
と実行して、rubyが使える状態になったかどうかを確認してください。
(-vというのはヴァージョンを確認するというオプションです。)

残念ながら現在のところWindows用だけで、Mac用はありません。

Windows 95, 98, MeでのPATHの設定方法

・まず、c:\ruby\bin\ruby.exeとなるようにファイルを解凍する。
→ここまでできない場合はちょっとつらいかもという感じがしますが、質問してみてください。

・c:\autoexec.bat というファイルがあります。これを編集します。
エクスプローラでc:\を表示してください。autoexecというファイルがあるはずなので、
それを右クリックして、編集を選びます。

・すでにいろいろファイルに書きこまれていると思いますが、それは消さないこと。
最後の行に次の一行を追加します。

set PATH=c:\ruby\bin;%PATH%

→現在のPATHに、c:\ruby\binを追加する、ということをしています。

・コンピュータを再起動します。

・コマンドプロンプトを出して、

c:\> ruby -v

といれたら、rubyからのメッセージが出るはずです。エラーメッセージが
出てきたら、なにか失敗してます。聞いてください。

Windows 2000, Windows XPでのPATHの設定方法

・マイ コンピュータを右クリック→「プロパティ(R)」
・「システムのプロパティ」が出たら、「詳細」をクリック
・「環境変数(E)...」をクリック
・「新規(N)...」をクリック
・変数名(N):のところに「 path 」
・変数値(V):のところに「 y:\ruby\bin;%path% 」
と入れる。以上のPATHは多摩美の場合で、
自宅の場合は「 c:\ruby\bin;%path% 」になる。
・「OK」「OK」「OK」
